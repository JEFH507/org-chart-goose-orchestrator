# Contributing

Thank you for contributing to goose-org-twin.

## Workflow
- Branching: feature branches per workstream (see README and docs/architecture)
- Commits: Conventional Commits (see docs/conventions/commit-style.md)
- PRs: Use the PR template, link relevant ADRs, include testing notes
- Reviews: 1+ reviewer, no force-push to shared branches

## Local-only files
Do not commit local secrets, tokens, or `.env.ce`. See `.gooseignore`.
