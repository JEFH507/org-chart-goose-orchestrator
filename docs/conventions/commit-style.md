# Conventional Commit Style

Use the following types:
- feat(scope): user-facing feature
- fix(scope): bug fix
- docs(scope): documentation only
- chore(scope): tooling/chores
- build(scope): build system or external dependencies
- ci(scope): CI configuration

Examples:
- feat(compose): add optional minio profile (docs only)
- docs(guides): clarify port overrides in dev-setup
- chore(scripts): add preflight ports placeholder

Scope guidance: use directory or component (e.g., compose, guides, scripts, api).
